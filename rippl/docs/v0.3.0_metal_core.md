# v0.3.0 Metal Core — Technical Documentation

## What This Release Is

v0.3.0 is the difference between rippl and a raw PyTorch loop.
v0.2.0 made rippl fast. v0.3.0 makes rippl correct.

Every feature in this release exists because standard neural network 
training fails on physics problems in a specific, documented way.
Each fix is mechanical, not heuristic.

## Causal Training

**The problem it solves:**
Standard PINNs optimize globally. The network sees t=0 and t=10 
simultaneously. Gradients from t=10 corrupt the solution at t=1 before 
t=1 has converged. This is why the wave equation failed 20+ times before 
causal training was implemented.

**The mechanism:**
Loss at time t_i is weighted by exp(-ε * Σ_{j<i} L_j).
High loss at early times suppresses gradient signal from late times.
The network is forced to learn the causal structure of the PDE.

**Usage:**
rp.run(..., causal=True, causal_mode="continuous")

**Proven result:**
Wave 1D L2: 8.77e-01 (without) → 8.38e-05 (with). 4 orders of magnitude.

## NTK Adaptive Loss Weighting

**The problem it solves:**
IC loss, BC loss, and PDE residual loss operate at different magnitudes.
A fixed weight of 100x for BCs works for heat but destroys NS training.
The optimizer gets confused by competing gradient directions.

**The mechanism:**
λ_i = mean(|∇L_total|) / (mean(|∇L_i|) + 1e-8)
Weights updated every 100 epochs via EMA smoothing.
Each loss term contributes proportionally to its gradient magnitude.

**Usage:**
rp.run(..., adaptive_loss=True, adaptive_loss_mode="gradient_norm")

**Proven result:**
NTK weighting: L2=6.80e-03 vs fixed weights: L2=2.04e-02. 3x improvement.

## Hard Boundary Conditions

**The problem it solves:**
Soft BC penalties (loss += 100 * ||u(boundary) - 0||²) can be ignored
by the optimizer when PDE residual dominates. Convergence depends on 
weight tuning, not physics.

**The mechanism:**
Ansatz: u(x) = g(x) + D(x) * ũ(x)
where g(x) satisfies BCs exactly and D(x) = 0 on boundary.
The network ũ(x) cannot violate BCs regardless of its output.

**Usage:**
rp.run(..., hard_bcs=True)  # auto-wraps model in AnsatzFactory

## Spectral Collocation

**The problem it solves:**
Random/Sobol sampling gives algebraic convergence O(N^{-1/d}).
For smooth PDEs on regular domains, this wastes compute.

**Chebyshev nodes:**
x_k = cos(kπ/N) — avoids Runge phenomenon, exponential convergence
for smooth solutions. Optimal for 1D/2D regular domains.

**Gauss-Legendre quadrature:**
Used when weak-form residual is needed. Exact integration of 
polynomials up to degree 2N-1. Required for hp-VPINN formulation.

**Usage:**
rp.run(..., collocation="chebyshev")  # or "legendre", "sobol", "random"
